Terms

  • Directory - Folder
  • Terminal or Command Line- Interface for Text Commands
  • CLI - Command Line Interface
  • cd - Change Directory
  • Code Editor - Word Processor for Writing Code (Visual Studio Code)
  • Repository - Project, or the folder/place where your project is kept
  • Github - A website to host your repositories online

Git Commands

  • status - Checks which files aren't tracked and which are tracked to commit.
  • clone - Bring a repository that is hosted somewhere like Github into a folder onto your local machine
  • add - Track your files and changes in Git
  • commit - Save your files in Git
  • commit -m - The -m stands for message, add another -m is for the description
  • commit -am - this is used when a file is already added and the file is modified (shortcut)
  • push - Upload Git commit to a remote repository, like Github
  • pull - Download changes from remote repository to your local machine, the opposite of push
  • remote -v - show which repositories are connected to this folder.
  • push -u origin master - this will set it so it pushes to the master branch, and afterwards you are able to use only push. (u = upstream)

Git Branching

                                                ↗ Hotfix Commit ↘
Commit 1 → Commit 2 → Commit 3 → Commit 4 → Commit 5
                ↘ Feature Commit 1 →   Feature Commit 2 ↗

  • Master branch - Commit 1 / Commit 2 / etc...
  • Feature branch - feature commit 1 / feature commit 2 / etc...
  • Hotfix branch - Hotfix Commit / etc..
A feature branch is usefull to test out different outcomes without changing the master branch (or main)
Eventually you have to merge it back to the main when you are done with your feature or hotfix.
But this is very usefull to use without screwing up your main project.

Git commands 1.1

  • git branch - This is to check which branch you are currently in and which branches there currently are, when there are muliple ones
  • git checkout - this is used to switch between branches
  • git checkout -b - this is used to make a new branch (feature-moving-image; example)
  • git branch -d - this is used to delete your branchname
  • git diff - this is used to see the changes that are made between files/ branches
  • git merge - this is used to merge with the main/master branch again
  • git push -u origin - This is used to add the new branch to your github repository
  • git pull - this is used to get the updated version from github when someone made changes and you dont have them yet
  • pull request - this is used on Github to merge is with the main or master branch
    inside this pull request you are able to add more comments on why you did something

Extra Info

If you dont have an upstream yet then you should use git pull -u origin branchname

Don't reuse branches that you already have used
If you've used one branch then you should remove the old branch with git branch -d branchname and afterwards create a new one for a next feature or hotfix

When a file is status : modified then you coud use a shortcut command
git commit -am "then add message here"

But when a file is newly created then you have to use git add first and git commit -m "message" to let it know
After you have done this you are able to use the shortcut git commit -am when the status says its modified again (this saves time)

Locally merge is used to update your feature branch with the master (Command line)
This is mainly used when you are working on a project and other people made a change and pushed it to the master and you dont have the updated version yet

Git Commands 1.2

  • git reset - to reset all of staged files
  • git reset *filename* - to reset the current staged filename
    example; git status
    staged files :
    • README.md
    • index.html
    git reset index.html
    This will remove the index.html from staging
    git status
    staged files :
    • README.md
  • git reset HEAD - undo the last commit you did
  • git reset HEAD~1 - undo the commit before your last commit
  • git log - is a log that shows all of your commits you did
    this is also displayed from latest to first
    example;
    • commit - hashcode
      Merge : if its merged
      Author : Who did the commit
      Date : Fri Feb 05 timestamp 2021 -somecode

      your commit message
    • commit - hashcode
      Author : Who did the commit
      Date : Fri Feb 03 timestamp 2021 -somecode

      your commit message
    • commit - hashcode
      Author : Who did the commit
      Date : Fri Feb 02 timestamp 2021 -somecode

      your commit message
    • commit - hashcode
      Author : Who did the commit
      Date : Fri Feb 01 timestamp 2021 -somecode

      your commit message etc etc..
    So if you want to return to a specific commit you could copy one of those specific hash codes
    This could be used when you did something and its not working good and you want to have the previous working version
  • git reset --hard - BE CAREFULL! this is a hard reset Where you delete all your previous commits and return to the specific hashcoded commit

Extra Info

Want to change things to a specific Github repository then you should use the fork symbol on Github
What this does ?
It basicly gives you a copy of that Github repository
BUT its under your name and you are the owner
So then you could change the code without screwing everyone over when you've made a change
